# Руководство пользователя

## Назначение приложения

Study Planner помогает студенту вести учебные задания, отслеживать дедлайны, статусы, приоритеты и дисциплины. Приложение поддерживает локальный кэш, поэтому ранее загруженные задания остаются доступными без сети.

## Вход и регистрация

1. Откройте приложение.
2. Введите email и пароль.
3. Для новой учетной записи переключитесь в режим регистрации.
4. Укажите ФИО и учебную группу.
5. После успешного входа откроется список заданий.

## Работа со списком заданий

На главном экране отображаются учебные задания пользователя.

Доступные действия:

- обновить список;
- найти задание по тексту;
- отфильтровать задания по дисциплине, статусу и приоритету;
- открыть детали задания;
- создать новое задание;
- увидеть признак локального кэша, если данные загружены из Room.

## Работа с дисциплинами

1. Откройте вкладку `Дисциплины`.
2. Введите название дисциплины.
3. При необходимости укажите преподавателя и семестр.
4. Нажмите `Добавить дисциплину`.

Созданные дисциплины становятся доступными в выпадающем списке при создании или редактировании задания. Существующую дисциплину можно отредактировать или удалить, если к ней не привязаны задания.

## Создание задания

1. Нажмите кнопку `+`.
2. Заполните название.
3. Добавьте описание.
4. Выберите дисциплину из списка.
5. Выберите приоритет: `Низкий`, `Обычный` или `Высокий`.
6. Выберите дату и время дедлайна кнопками `Дата` и `Время`.
7. Нажмите `Сохранить`.

## Детали задания

Экран деталей показывает:

- название;
- дисциплину;
- дедлайн;
- приоритет;
- статус;
- описание.

Доступные действия:

- отметить задание как выполненное;
- перевести задание в работу;
- добавить напоминание;
- удалить напоминание;
- открыть редактирование;
- удалить задание.

## Профиль

Экран профиля показывает email, ФИО и группу пользователя. Здесь доступен выход из учетной записи.

## Оффлайн-режим

Если сервер временно недоступен, приложение показывает данные из Room-кэша. Оффлайн-режим работает после первого успешного онлайн-входа: приложение сохраняет JWT-токен, данные текущего пользователя и ранее загруженные задания/дисциплины. При следующем запуске без backend приложение восстанавливает сохраненную сессию и открывает список заданий из локального кэша. Некоторые изменения заданий сохраняются в очередь pending-операций и повторяются при нажатии кнопки обновления на экране списка заданий.

## Ошибки

Если действие выполнить нельзя, приложение показывает причину ошибки. Например, при неверном семестре будет показано сообщение о допустимом диапазоне, при выключенном backend - сообщение о недоступности сервера, а при истекшей сессии - просьба войти заново.
