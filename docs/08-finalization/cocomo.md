# Оценка трудозатрат COCOMO

## Исходные данные

Проект относится к органическому типу COCOMO: индивидуальная разработка, понятная предметная область, ограниченный размер команды, типовая архитектура.

| Параметр | Значение |
|---|---:|
| Оценка размера backend | 2.8 KLOC |
| Оценка размера mobile | 2.2 KLOC |
| Документация и конфигурация | 1.0 KLOC эквивалента |
| Итого | 6.0 KLOC |

## Формула COCOMO Basic

Для organic mode:

```text
Effort = 2.4 * KLOC^1.05
Time = 2.5 * Effort^0.38
People = Effort / Time
```

## Расчет

```text
Effort = 2.4 * 6.0^1.05 = 15.8 person-month
Time = 2.5 * 15.8^0.38 = 7.1 months
People = 15.8 / 7.1 = 2.2 people
```

## Интерпретация для курсового проекта

Так как проект выполняется индивидуально и часть решений является учебной, фактическая реализация ограничивается готовой учебной версией:

- основные серверные сценарии;
- мобильный клиент с ключевыми экранами;
- offline-cache как read-through кэш с pending-операциями;
- документация и защита.

Оценка COCOMO показывает, что полноценная промышленная версия потребовала бы команды из 2-3 человек на несколько месяцев. Курсовая версия реализует управляемый объем в рамках семестра.
