# Этап 3. Проектирование базы данных

## Цель этапа

Спроектировать серверную базу данных для мобильной информационной системы планирования учебных заданий студентов: определить логическую ER-модель, физическую структуру таблиц, ключи, ограничения, индексы и стратегию отображения JPA Entity на таблицы PostgreSQL.

## Основные проектные решения

- PostgreSQL используется как основное хранилище актуальных данных серверной части.
- Все пользовательские данные привязаны к владельцу через `users.id`, чтобы сервер мог изолировать данные студентов.
- Учебные задания связаны с дисциплинами, напоминаниями и вложениями.
- Справочные состояния задания и приоритеты реализуются через enum-поля с `CHECK`-ограничениями.
- Для быстрого списка заданий предусмотрены индексы по пользователю, дисциплине, дедлайну, статусу и приоритету.
