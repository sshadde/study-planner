# Оффлайн-режим

## Назначение

Оффлайн-режим позволяет студенту видеть ранее загруженные задания и дисциплины без доступа к серверу. При ошибке сети repository возвращает данные из Room-кэша.

## Room-таблицы

| Таблица | Kotlin Entity | Назначение |
|---|---|---|
| `assignments` | `LocalAssignment` | Кэш учебных заданий |
| `courses` | `LocalCourse` | Кэш дисциплин |
| `pending_operations` | `PendingOperation` | Очередь локальных изменений |

## Алгоритм чтения списка заданий

```text
1. ViewModel вызывает AssignmentRepository.getAssignments(filter).
2. Repository пытается получить данные через Retrofit.
3. Если сервер доступен, результат сохраняется в Room и показывается пользователю.
4. Если сервер недоступен, выполняется поиск по Room-кэшу.
5. UI получает состояние `offline=true` и показывает признак локального кэша.
```

## Алгоритм локальных изменений

```text
1. Пользователь создает, изменяет, удаляет задание или меняет статус.
2. Repository пытается выполнить REST-запрос.
3. Если запрос не удался, операция сохраняется в pending_operations.
4. AssignmentSyncManager позже повторяет pending-операции.
```

## Реализованные компоненты

| Компонент | Назначение |
|---|---|
| `AssignmentDao` | Поиск, сохранение и удаление локальных заданий |
| `CourseDao` | Сохранение и чтение дисциплин |
| `PendingOperationDao` | Очередь операций для синхронизации |
| `AssignmentRepository` | Выбор между сервером и кэшем |
| `AssignmentSyncManager` | Повторная отправка pending-операций |

## Ограничения

Оффлайн-режим не заменяет серверное хранилище актуальных данных. Приложение кэширует задания и дисциплины, а операции создания, изменения, смены статуса и удаления заданий сохраняет в очередь `pending_operations` и повторяет при следующей синхронизации.
