# Функциональные и нефункциональные требования

## Функциональные требования

| ID | Требование | Приоритет |
|---|---|---|
| FR-01 | Система должна позволять пользователю зарегистрироваться по email и паролю | Высокий |
| FR-02 | Система должна позволять пользователю войти в приложение и получить JWT-токен | Высокий |
| FR-03 | Система должна отображать список учебных заданий текущего пользователя | Высокий |
| FR-04 | Система должна позволять создавать учебное задание | Высокий |
| FR-05 | Система должна позволять редактировать учебное задание | Высокий |
| FR-06 | Система должна позволять удалять учебное задание | Средний |
| FR-07 | Система должна позволять изменять статус задания | Высокий |
| FR-08 | Система должна поддерживать поиск и фильтрацию заданий | Средний |
| FR-09 | Система должна позволять управлять дисциплинами | Средний |
| FR-10 | Система должна позволять создавать напоминания о дедлайнах | Средний |
| FR-11 | Система должна позволять просматривать и удалять напоминания | Средний |
| FR-12 | Мобильное приложение должно сохранять задания и дисциплины в локальном кэше | Высокий |
| FR-13 | Мобильное приложение должно отображать кэшированные данные без подключения к сети | Высокий |
| FR-14 | Сервер должен предоставлять OpenAPI-документацию | Высокий |

## Нефункциональные требования

| ID | Требование | Критерий проверки |
|---|---|---|
| NFR-01 | Серверная часть должна быть реализована на Java 17 и Spring Boot | Наличие backend-проекта и сборки |
| NFR-02 | Мобильное приложение должно быть реализовано на Kotlin с Jetpack Compose | Наличие Android-проекта |
| NFR-03 | Система должна использовать PostgreSQL для серверного хранения данных | Конфигурация datasource и миграции/DDL |
| NFR-04 | Пароли должны храниться в виде BCrypt-хеша | Проверка реализации security-слоя |
| NFR-05 | Защищенные API должны требовать JWT-токен | Тесты и проверка Swagger/Postman |
| NFR-06 | Покрытие серверной части модульными тестами должно быть более 40% | Отчет JaCoCo |
| NFR-07 | Архитектура backend должна соответствовать PCMEF | Проверка структуры пакетов |
| NFR-08 | API должно возвращать корректные HTTP-статусы | Тесты контроллеров |
| NFR-09 | Пользовательский интерфейс должен иметь минимум 5 экранов | Скриншоты и демонстрация приложения |
| NFR-10 | Приложение должно обрабатывать состояния загрузки, ошибки и пустого списка | Проверка UI-сценариев |
