# Этап 1. Проектирование требований

## Цель этапа

Перевести бизнес-модель в системные требования: определить пользователей системы, функциональные возможности, ограничения, системные прецеденты, предметную модель и связи между бизнес-требованиями и функциями приложения.

## Предварительный перечень системных функций

- Регистрация пользователя.
- Аутентификация пользователя через JWT.
- Просмотр списка учебных заданий.
- Создание учебного задания.
- Редактирование учебного задания.
- Удаление учебного задания.
- Просмотр деталей задания.
- Изменение статуса задания.
- Поиск и фильтрация заданий.
- Управление дисциплинами.
- Создание напоминаний.
- Просмотр заданий из локального кэша при отсутствии сети.
- Синхронизация данных между мобильным приложением и сервером.
