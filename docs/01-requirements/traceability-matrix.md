# Таблица трассировки требований

## Назначение

Таблица связывает бизнес-требования, системные функции, элементы предметной модели и планируемые API-эндпоинты. Это позволяет проверить, что каждая важная бизнес-потребность получила отражение в проектируемой системе.

| ID | Бизнес-требование | Системное требование | Use Case | Сущности | API / компонент |
|---|---|---|---|---|---|
| BR-01 | Студент должен хранить задания в едином месте | Система должна предоставлять CRUD для учебных заданий | UC-03, UC-05, UC-06, UC-07 | Assignment, Course, User | `/api/assignments` |
| BR-02 | Студент должен видеть ближайшие дедлайны | Система должна сортировать и фильтровать задания по сроку | UC-03, UC-09 | Assignment | `/api/assignments` |
| BR-03 | Студент должен отслеживать выполнение заданий | Система должна поддерживать статусы заданий | UC-08 | Assignment, AssignmentStatus | `/api/assignments/{id}/status` |
| BR-04 | Студент должен планировать напоминания | Система должна позволять создавать, просматривать и удалять напоминания | UC-11 | Reminder, Assignment | `/api/assignments/{id}/reminders`, `/api/reminders/{id}` |
| BR-05 | Пользовательские данные должны быть защищены | Система должна поддерживать JWT-аутентификацию и роли | UC-01, UC-02 | User, UserRole | `/api/auth/register`, `/api/auth/login` |
| BR-06 | Студент должен видеть только свои задания | Сервер должен фильтровать данные по текущему пользователю | UC-03, UC-04 | User, Assignment | Security context, repositories |
| BR-07 | Система должна работать при временном отсутствии сети | Мобильное приложение должно кэшировать задания и дисциплины | UC-13, UC-14 | Assignment, Course | Room, sync service |
| BR-08 | Студент должен группировать задания по дисциплинам | Система должна поддерживать управление дисциплинами | UC-10 | Course, Assignment | `/api/courses` |
| BR-09 | Студент должен быстро находить нужное задание | Система должна поддерживать поиск и фильтры | UC-09 | Assignment, Course | `/api/assignments` |
| BR-11 | Система должна быть понятной для проверки и развития | API должно иметь документацию OpenAPI | Все API use cases | Все сущности | Swagger UI |
| BR-12 | Реализация должна соответствовать курсовой методике | Проект должен использовать PCMEF и разделение слоев | Все use cases | Все сущности | presentation, control, mediator, entity, foundation |
