# Бизнес-контекст

## Описание предметной области

Предметная область проекта связана с организацией учебной деятельности студентов. В течение семестра студент получает задания по разным дисциплинам: лабораторные работы, практические задания, курсовые этапы, тесты, доклады и самостоятельные работы. Каждое задание имеет срок выполнения, приоритет, статус, описание требований и связь с дисциплиной.

Текущий процесс часто зависит от личной дисциплины студента и не поддерживается единой цифровой системой. Информация о заданиях может храниться в нескольких местах, из-за чего студенту сложно быстро понять, какие задачи являются срочными, какие уже выполнены, а какие требуют уточнения.

## Проблемы текущего процесса

- Разрозненное хранение информации о заданиях.
- Риск пропуска дедлайнов.
- Отсутствие единого списка задач по всем дисциплинам.
- Сложность приоритизации учебной нагрузки.
- Невозможность быстро посмотреть задания без доступа к интернету.
- Отсутствие прозрачной истории изменения статусов.

## Предлагаемое решение

Мобильное приложение с серверной частью позволит студенту централизованно управлять учебными заданиями. Система будет поддерживать регистрацию и вход, список дисциплин, создание и редактирование заданий, фильтрацию, поиск, статусы выполнения, напоминания и локальное кэширование.

## Основные бизнес-процессы

| Процесс | Краткое описание |
|---|---|
| Регистрация студента | Создание учетной записи и профиля студента |
| Планирование задания | Добавление задания с дисциплиной, сроком, приоритетом и описанием |
| Контроль выполнения | Изменение статуса задания и просмотр прогресса |
| Поиск и фильтрация | Быстрый поиск задач по дисциплине, сроку, статусу и приоритету |
| Напоминание о сроках | Создание записей-напоминаний о приближающихся дедлайнах |
| Работа без сети | Просмотр сохраненных заданий из локального кэша |
