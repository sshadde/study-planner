# SWOT-анализ

| Сторона | Описание |
|---|---|
| Strengths / Сильные стороны | Мобильный доступ к заданиям; единое место хранения информации; поддержка статусов, приоритетов и напоминаний; оффлайн-доступ к кэшированным данным |
| Weaknesses / Слабые стороны | Необходимость поддержки серверной и мобильной частей; ограниченный срок разработки; первая версия не интегрируется с официальными системами университета |
| Opportunities / Возможности | Интеграция с LMS и расписанием; push-уведомления; аналитика учебной нагрузки; совместное планирование для учебных групп |
| Threats / Угрозы | Ошибки синхронизации; потеря актуальности данных при ручном вводе; недостаточное тестирование; возможное усложнение требований |

## Вывод

Проект имеет выраженную практическую ценность, так как решает частую проблему студентов: потерю контроля над учебными заданиями и сроками. Основной риск связан со сложностью клиент-серверной реализации, поэтому разработку следует вести итерационно: сначала серверное ядро и основные сценарии, затем мобильный интерфейс и оффлайн-кэш.
