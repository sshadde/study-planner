# Паспорт проекта

## 1. Общая информация

**Название проекта:** Разработка мобильного приложения для информационной системы планирования учебных заданий студентов.

**Траектория:** мобильная разработка.

**Автор:** Петрищев Ярослав Дмитриевич.

**Плановая длительность:** 18 недель.

## 2. Бизнес-контекст

Студенты часто используют разрозненные инструменты для фиксации учебных заданий: заметки, мессенджеры, таблицы, бумажные записи и устные напоминания. Из-за этого возрастает риск пропуска дедлайнов, потери информации о требованиях к заданию и неравномерного распределения учебной нагрузки.

Разрабатываемая система предлагает единое мобильное приложение для планирования учебных заданий. Студент сможет хранить задания по дисциплинам, отслеживать сроки и статусы, получать напоминания, фильтровать задачи и иметь доступ к актуальным данным даже при временном отсутствии сети.

## 3. Цели проекта

- Разработать мобильное приложение для планирования и контроля учебных заданий.
- Реализовать серверную часть на Java Spring Boot с REST API и JWT-аутентификацией.
- Обеспечить локальное кэширование данных для работы в оффлайн-режиме.

## 4. Ключевые показатели успеха

| Показатель | Целевое значение |
|---|---|
| Количество экранов мобильного приложения | Не менее 5 |
| Количество REST-эндпоинтов | Не менее 8 |
| Покрытие серверной части модульными тестами | Более 40% |
| Поддержка оффлайн-режима | Кэширование заданий, дисциплин и статусов |
| Документация API | Swagger UI / OpenAPI |
| Соответствие PCMEF | Разделение control, mediator, entity, foundation |

## 5. Ключевые риски

| Риск | Влияние | Стратегия реагирования |
|---|---|---|
| Недостаток времени на мобильный клиент | Высокое | Сначала реализовать основной сценарий: список, детали, создание и статус задания |
| Сложность синхронизации оффлайн-данных | Среднее | Реализовать read-through кэш и очередь локальных изменений для повторной отправки |
| Ошибки в архитектурном разделении PCMEF | Высокое | Фиксировать зависимости через пакеты, интерфейсы сервисов и репозиториев |
| Недостаточное тестовое покрытие | Среднее | Писать тесты для сервисов, валидаторов и контроллеров параллельно с реализацией |
