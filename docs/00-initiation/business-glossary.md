# Бизнес-глоссарий

| Термин | Определение |
|---|---|
| Студент | Пользователь системы, планирующий и отслеживающий учебные задания |
| Учебное задание | Работа, которую студент должен выполнить в рамках дисциплины |
| Дисциплина | Учебный предмет, к которому относится задание |
| Дедлайн | Крайний срок выполнения учебного задания |
| Приоритет | Степень важности или срочности задания |
| Статус задания | Текущее состояние задания: новое, в работе, выполнено, просрочено |
| Напоминание | Уведомление о приближении срока выполнения задания |
| Профиль студента | Набор данных о студенте: имя и учебная группа |
| Оффлайн-режим | Возможность работать с сохраненными данными без подключения к сети |
| Синхронизация | Обновление локальных и серверных данных до согласованного состояния |
| REST API | Интерфейс обмена данными между мобильным клиентом и сервером |
| JWT-токен | Токен, подтверждающий подлинность пользователя при запросах к API |
| Кэш | Локальная копия данных, сохраненная на мобильном устройстве |
| Фильтр | Условие отбора заданий по статусу, дисциплине, сроку или приоритету |
| Поиск | Механизм нахождения заданий по текстовому запросу |
| Учебная нагрузка | Совокупность заданий, которые студент должен выполнить за период |
| План на неделю | Представление заданий, сгруппированных по ближайшим датам |
| Просроченное задание | Задание, срок которого истек, а статус не равен «выполнено» |
| Архив задания | Набор завершенных или неактуальных заданий, скрытых из основного списка |
